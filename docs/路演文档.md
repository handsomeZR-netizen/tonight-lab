## 一、项目概览

- **项目名称**：Tonight Lab（今晚不再纠结了）
- **介绍**：把"今晚到底做什么"包装成 AI 场景叙事的沉浸式信息流——用户继续刷信息流，AI 情境卡直接在原地给出可比较、可调整、可保存的下一步。

---

## 二、为什么做（Why）

### 1. 受众是谁？

**核心受众**：22–35 岁的城市青年，主要画像：

- **身份**：互联网/媒体/设计/学生群体，独居或合租，工作日晚上 7–11 点处于"低能量决策窗口"。
- **数字习惯**：日均短视频时长 ≥ 90 分钟（抖音/小红书/B 站轮换），手机里同时装着美团、大众点评、高德、咕咚、Keep、网易云等 6+ 个工具型 app。
- **审美偏好**：杂志/电影感、editorial 视觉、克制的微交互；不喜欢"恐吓式 CTA"和"信息密度爆炸"的传统 dashboard。
- **遇见情境**：晚饭前的通勤地铁里、洗澡前躺床上、周末早起咖啡时段——他们已经在刷信息流，不会"专门打开一个工具"。

### 2. 你想为他们创造什么？

**想解决的核心痛点**：**小决策疲惫（micro-decision fatigue）**——人不是缺信息，而是缺"现在就能落地的下一步"。

**两个真实观察**：

> **观察 1：22:14 的麻辣烫**
> 一个朋友在群里发"今晚吃啥呢"，群里 7 个人接力推荐了 14 家店、3 个 app 链接和 2 张菜单截图。一小时后她说"算了我点了麻辣烫"。她不是不知道该吃啥，她是被"应该吃顿好的"和"今晚不想再做决定"两股力量拉扯，最后选了一个零思考成本的兜底。

> **观察 2：周日的"等什么也没等到"**
> 另一个朋友周日下午发朋友圈："本来想出门走走，刷了半小时小红书，看完 8 个 city walk 推荐，最后躺床上刷完了一部剧。" 信息流给了她"应该出门"的氛围感，但没人帮她把"应该"压缩成"现在去坐 17 号线到南京西路下，步行 12 分钟到武康路 376 号"。

**想传递的体验**：让用户在不退出信息流的前提下，**完成一次小决定**——保留刷的轻松感，加上工具的确定性。整体氛围目标：编辑过的、克制的、像翻一本"今晚专辑"。

### 3. 现有体验缺什么？

调研了 4 类同质方案的盲点（自查 + 5 位目标用户深访）：

| 同类 | 缺什么 |
|---|---|
| **短视频信息流**（抖音/小红书） | 给氛围，不给"然后呢"。看完 20 个 vlog 还要自己开高德搜地址、看营业时间、估价格——决策成本被推回给用户。 |
| **传统 LBS 工具**（美团/大众） | 给清单，不给立场。给 50 家店让你自己排序——选择悖论加剧，而不是缓解。 |
| **AI Chat 类应用**（ChatGPT/各家小助手） | 对话式回答和"刷"的节奏不兼容；要键盘、要思考问什么、要复制粘贴下一步。 |
| **生活方式订阅号**（公众号长文） | 给故事，不给可执行性。看完很爽，关掉就忘。 |

**共同问题**：所有方案都让用户**离开当下的浏览**去另一个工具完成决策。真正的痛点不是缺信息，是缺"不打断浏览的执行层"。

> 用户访谈金句（5 位中的 3 位都说过类似的话）：
> "我不是要更多推荐，我是要别再让我自己做最后一步。"

### 核心设计功能（共 3 个）

---

#### 功能 1 · AI 情境卡（信息流原生层）

**功能名称**：AI Scenario Cards · 信息流里的轻决策层

**设计意图 / 解决的问题**：

把"用工具"这件事压扁成"刷信息流时多看一张卡"。卡片承接用户当下的兴趣（刚刷完一段火锅探店视频 → 出现"今晚吃什么"情境卡），不要求用户切换上下文。每张卡片做三件事：

- **整理偏好**：把用户散落的预算/口味/时间/情绪收成 3 个可比较选项（而非 50 家店）
- **解释原因**：每个选项写明"为什么是它"（信任迁移到 AI 推理上，不是黑盒）
- **就地操作**：保存、刷新、调整、展开都在卡片内完成，不跳出信息流

四个场景：**今晚吃什么 / 3 小时城市逃逸 / 赛前 15 分钟 / 下班回血**。每个场景的卡片有独立的 tone（暖橙/雾蓝/深绿/紫调）和专属交互（菜单翻牌、路径绘制、战术板、呼吸气泡）。

**关键截图 / 流程**：

[文生图占位符：1080×1920 竖屏手机模拟图。上半部分是抖音/小红书风格的短视频信息流（火锅探店视频缩略图、热度数字），中间无缝插入一张 Tonight Lab 情境卡——卡片顶部是温暖橙色的"今晚吃什么"封面图（一碗冒着热气的牛肉面、暖色厨房光），中间是 AI 标签"结合你的预算和今晚的胃口"，下方三个可比较选项（番茄牛腩饭 ¥36 / 越南牛肉粉 ¥38 / 小碗云吞面 ¥28）每个都有迷你卡片视觉，底部是深色"打开完整玩法"主按钮加箭头。整张卡片有微妙的纸张质感和暖光晕，像被精心编辑过的杂志页]

[文生图占位符：横屏对比图。左侧标"传统信息流体验"——用户看完短视频后跳到 5 个不同 app（美团、点评、高德、外卖小程序、备忘录），有混乱的转场箭头，配色冷灰。右侧标"Tonight Lab 体验"——用户全程在信息流里完成：刷到 → 卡片展开 → 选择 → 保存，单一容器、暖色调、流畅过渡。中间一条分隔线写「同样的 60 秒，不同的决策成本」]

#### 功能 2 · Editorial 详情页 · 把"打开 app"变成"翻一本专辑"

**功能名称**：Editorial Detail Page · 杂志式可交互详情

**设计意图 / 解决的问题**：

如果用户决定深入了解某张卡片，传统体验是跳到一个功能型详情页（参数列表、地图、评分）。我们换一种叙事：**把详情页做成一期杂志**。

每个场景对应一本"Issue"（吃什么 = Issue 01 · MENU LAB / 城市逃逸 = Issue 02 · CITY FIELD GUIDE / 赛前 = Issue 03 · MATCH ROOM / 回血 = Issue 04 · AFTER HOURS）。

页面元素：

- **Pull quote**：editorial 级别的大字号引文，定调当晚的情绪
- **Stat block**：实时跟随用户调节的核心指标（预算/辣度/饱腹感百分比）
- **Field notes**：编辑笔记体的"为什么这样设计"小段
- **WebGL 氛围层**：每个 tone 有专属 Three.js 场景（暖雾、城市光流、赛场能量场、呼吸光晕）
- **Signature interaction**：每个场景有一个标志性交互（吃什么 = 菜单翻牌；城市逃逸 = SVG 路径绘制时间线；赛前 = X/O 战术板；回血 = 呼吸气泡）

不是为了炫技，是为了让"翻这一页"本身就是一次轻量的回血/放松/兴奋——内容和情绪共同承载决策。

**关键截图 / 流程**：

[文生图占位符：详情页"今晚吃什么"实验台截图。1440×900 桌面视图。左侧：大号衬线字体（Fraunces italic）的标题"今晚吃什么实验台"，下面一段 editorial 风格的副标，再下方是"实验台控制面板"——预算/辣度/速度三组 pill 控件 + 一个饱腹感 0-100% 的滑块。右侧：实时跟随调整的"今晚下单结论"卡片（深色背景、Sparkles 图标、大字标题）。整体配色暖橙 + 米白，背景叠了一层 WebGL 暖雾（轻微动态颗粒和暖色光斑）。顶部有杂志式 meta 行（Date / Section / Budget / Edition № 01）]

[文生图占位符：详情页"3 小时城市逃逸"截图。1440×900 桌面视图。冷蓝黄昏调，SVG 路径绘制时间线从左上"咖啡馆 16:30"画到右下"地铁回家 19:20"，沿途几个站点用极简圆点标记，每个站点配一张小图片预览（街角咖啡、河边、独居小酒馆）。右侧栏是"今晚路线"指标卡：步行压力·低 / 逃离工位感·78 / 社交耗电·12%。整体像 The New York Times 的旅行版面]

[文生图占位符：4 张详情页的封面拼贴。等比例 16:10 矩阵，4 张图分别用 4 个 tone 主色（暖橙 / 冷蓝 / 深绿 / 紫调）的渐变背景，叠上各自的英文 issue 标题（TONIGHT'S MENU LAB / MICRO ESCAPE FIELD GUIDE / PRE-MATCH ROOM / AFTER HOURS RITUAL），右上角各自的 Issue 编号 01/02/03/04，下方有中文副标。像四张 OG 分享卡或杂志封面墙]

#### 功能 3 · Tone 主题系统 + 持久化收藏

**功能名称**：Tone System & Persistent Saves · 让作品记住"今晚的你"

**设计意图 / 解决的问题**：

信息流的天然弱点是**无痕迹**——刷过就忘，再回来什么都不剩。Tonight Lab 想要做的"轻决策"不是一次性的，需要在多次使用之间形成**情绪连续性**和**个人沉淀**。

两层机制：

- **Tone 切换器**：落地页 hero 区放一个 4-pill 切换器（暖橙/雾蓝/深绿/紫调），用户在第一秒就能选择"今晚我处于什么情绪"。切换后 hero 背景、手机预览光晕、甚至首张推荐卡的色调都跟着变。
- **持久化收藏**：在任何一张卡片上点书签，写入 localStorage。下次回来时收藏数会在 hero 顶部显示；进入某个 tone 的详情页时，如果之前收藏过对应卡片，顶部会出现一条细 ribbon "你今晚已收藏"。

它不是"我的收藏"列表（那太像传统 app），它是**让作品记得你**——下次刷到 Tonight Lab，它会用你上次选择的 tone 打开，用一条 ribbon 告诉你"你回来了"。

附带的工程层细节：

- 全键盘可达（↑↓ 切卡 / Esc 回顶 / Tab 显示 skip-to-main）
- feed → detail 的 View Transitions（原生浏览器 API，封面图 morph 过渡）
- Layered scroll motion（每个 section 进入视口时 stagger fade-in）

**关键截图 / 流程**：

[文生图占位符：落地页 hero 截面，重点展示 tone 切换器交互。1440×900 桌面视图。左侧大号 Fraunces 衬线标题"AI 情境卡，把下一步放回信息流。"，标题上方有一行 sparkles 标签 + 一个 4-pill tone 切换器（4 个小圆点 + 中文标签：今晚吃什么 / 城市逃逸 / 赛前观察 / 回血），其中"今晚吃什么"被激活——金色 ring 边框。整段 hero 背景叠了一层暖橙渐变光晕，右侧是 390px 宽的手机模拟器外壳，里面正在显示一张暖色"今晚吃什么"情境卡。整张图配色温暖，光晕有微妙的呼吸感]

[文生图占位符：动态展示 tone 切换前后的对比。左右分屏，左半"food / 暖橙 mode"——背景暖米色 + 手机内是麻辣火锅卡；右半"recovery / 紫调 mode"——背景冷紫 + 手机内是夜灯呼吸气泡卡。中间用一个细横条箭头连接，标"一次切换 · 整页氛围迁移"]

[文生图占位符：详情页顶部 "你今晚已收藏" ribbon 的特写。横长条，1200×120。左侧是 BookmarkCheck 图标 + 文字"你今晚已收藏 · 11.16"，背景用对应 tone 的浅色（如 amber-50），右侧有日期戳。整条 ribbon 紧贴详情页 hero 顶部、BackToFeedButton 旁边，存在感低但传达了"作品记得你"的细节]

### 工程实现 & 性能（路演备用）

- **技术栈**：Next.js 15 App Router · React 19 · TypeScript strict · Tailwind 3 · Framer Motion 12 · Three.js + React Three Fiber · Zod
- **页面性能**（Lighthouse mobile，部署后实测）：
  - `/` → Performance 89 · Accessibility 100 · SEO 100 · Best Practices 100
  - `/cards/food` → 87 / 100 / 100 / 100
  - `/press` → 95 / 96 / 100 / 100
- **架构亮点**：落地页全 RSC（首屏 JS bundle 仅 20.5 kB）；OG 图通过 `next/og` ImageResponse 在 Edge runtime 动态生成；favicon 同样动态；4 个详情页静态预渲染；WebGL scene 动态导入并 gate 在 `useWebGLSupport`；完整的 error/loading/not-found segment 兜底；JSON-LD 结构化数据（SoftwareApplication + Article）；安全头（X-Content-Type-Options / Referrer-Policy / Permissions-Policy）；robots.txt + sitemap.xml 全部自动生成。

## 四、未来规划（如果晋级，下一步怎么做）

### 近 3 个月（11.16 – 02.16）

**1. 内容层：接入真实推荐能力**
- 用 LLM（Claude / GPT）替换当前的 mock 数据，把"今晚吃什么"的 3 个候选项做成真请求：传入用户地点、时间、预算偏好、当下情绪 → 返回结构化 card schema
- 接美团/大众点评开放 API（部分数据集）/ 大众化的地理位置 SDK
- 在 4 个现有场景基础上加 2 个：**周末早晨**（咖啡 / 早午餐路线）和 **失眠夜**（10 分钟可执行的睡前流程）

**2. 体验层：让原型可被"真用一周"**
- PWA 化，让用户加到桌面，next-pwa 接管离线缓存
- 推送通知：每天 18:30 推一张"今晚卡"，可选 tone
- 收藏从 localStorage 迁到 Supabase（匿名 device id 即可，无需登录）
- 加分享：把任意一张卡导出为 1080×1350 的小红书风格图（用现有 OG 模板扩展）

**3. 验证层：跑一轮真实用户测试**
- 招募 20 位目标用户，连续 7 天试用
- 衡量指标：**单次决策时长**（从打开到产生行动）、**收藏复看率**、**tone 切换分布**
- 输出一份"决策疲惫"的 mini whitepaper，配 demo 一起发

### 中长期（1 年内）

**1. 平台化：把"卡片层"做成可被嵌入的能力**
- 不一定要做独立 app。和小红书/B 站/抖音的内容创作者合作：你发一条火锅探店视频 → 视频卡片下方自动生成 Tonight Lab 情境卡（"今晚我要吃同款"）
- SDK 形态：iframe 或 web component，内容平台调用一行代码即可嵌入

**2. 编辑共创：不只让 AI 写，让真编辑参与**
- 每周一期 editorial issue，由真人编辑 + AI 协作生产
- 类比：The Browser / Brain Pickings 的"小决策版"
- 用户付费订阅或免费但带品牌 issue

**3. 跨平台扩展**
- iOS / Android 原生包壳（基于 Capacitor 或 Expo）
- Apple Watch 微卡片（"今晚吃什么" 三选项直接戴在手腕上）
- 小程序版本（微信生态触达 1.3B）

### 商业化或落地设想

**Plan A · 内容平台合作（最低门槛）**
- 给内容平台提供"决策卡片"能力，按调用次数收费或抽成
- 切入点：餐饮 / 旅游 / 运动 / 健康四个垂类，每个垂类品牌愿意付费"出现在用户的今晚卡里"
- 商业模式：CPC（每次卡片被打开收费）+ CPS（用户通过卡片下单分成）

**Plan B · 品牌 Issue（高溢价、低频次）**
- 和品牌联合出 editorial issue（例：和星巴克合作"周日上午咖啡专辑" / 和 Nike 合作"赛前 15 分钟" issue）
- 单 issue 收取品牌赞助费，产出内容资产同时拥有品牌曝光
- 类比：苹果 Today at Apple、星巴克 Reserve 的内容化合作

**Plan C · 订阅制（长期 IP）**
- 用户付费订阅"每周一期 Tonight Lab Issue"，年订 ¥99
- 解锁更深度的 editorial 内容 + 个人化 AI 推荐 + 收藏 / 分享高级模板
- 类比：The New York Times Cooking、Pitchfork Reviews

**优先验证顺序**：A → B → C（先证明能给品牌带流量，再做品牌深度合作，最后用户付费）
